# CONTENT_GUIDE — 假数据风格指南

> 给生成阶段的 `mockData.ts` / `pickImage(seed)` 提供「调子」约束。目的是让 demo 在没有真实数据时仍然像一个中国大屏 TV 产品的真实截屏，而不是 lorem ipsum 拼出来的样品。

## 1. 标题与节目名

- **现实题材但虚构**。不抄真实 IP（不出现「狂飙」「漫长的季节」），也不写英文 lorem。
- **常用前缀**：地理（南岭 / 长江 / 漠河 / 嘉陵）、自然（夜行 / 入海 / 风雪 / 雾下）、人事（旧影 / 回声 / 灯塔 / 三月）。
- **节目命名公式**：`<地理或时令> + <动词或意象>`；2–5 字最稳，6 字以上要确认布局能放下。
- **示例**：南岭夜行 · 长江入海 · 城与火 · 风过塞外 · 南国旧影 · 雾下三章 · 嘉陵旧事 · 北纬以南
- 综艺类可稍轻：`厨房里的家` · `周日散步` · `小镇周末`。
- 音乐类不抄真实歌名，编虚的：`想去海边的人` · `夜色温柔` · `给你一瓶魔法药水`。

## 2. 人名与角色

- **避开真实公众人物**（除非该页面就是「真实人物专题」且有授权图）。
- 用常见复姓 / 单姓 + 通用名：周宁、李文哲、陈白露、苏屿、Jiang Wei。
- 主持人 / 嘉宾用「姓 + 单字 + 头衔」：「林川 主播」「沈书 嘉宾」。

## 3. 时间戳

- 一律用相对今天的真实分布，不要 1970 / 2099。
- TV 节目「今晚 19:30」「明天 14:00」「周六 20:00」最自然。
- 历史回顾用「2025-09」「2024 春」级别精度，不写完整 ISO。
- 时长用 `45 分钟` / `1 小时 20 分` / `12 集` / `36 集`。

## 4. 评分

- 集中在 **7.5–9.5**，避免 6.x（太可疑）和 9.9（太假）。
- 评分用 1 位小数：`8.7` `9.1`。
- 配 `XXX 万人在看` 当辅助 meta，数字用 `2.3 万` / `18.6 万` / `45 万`。

## 5. 计数与时长

- 集数：`8 / 12 / 16 / 24 / 32 / 36 / 48` 这几个挡位，少出 `13` `17` 这类不自然数。
- 歌单：`24 首` / `32 首` / `48 首` / `68 首`。
- 历史观看：`观看至第 6 集` / `还有 12 分钟` / `已看完`。

## 6. 文案语气

- DUI / AI 助手回复用「平和 + 给方向」，不要「我已为您安排…!」感叹号。
- 推荐位副标题 8 字内，结构 `<时段或题材> · <亮点>`，例：`今晚首播 · 自然纪录` · `家庭剧场 · 治愈向`。
- 角标只用既定文案集：`VIP` / `免费` / `首播` / `补集` / `4K` / `HDR` / `AI陪看`。

## 7. 类别与标签

- 影视：纪录片 / 剧集 / 综艺 / 电影 / 短视频 / 频道 / 回看。
- 音乐：华语 / 古典 / 治愈 / 经典 / OST / 儿歌。
- 人物：导演 / 演员 / 主持 / 歌手 / 嘉宾。
- 避免 `Other` `Misc` `Test` 这种系统味标签。

## 8. 图片 seed 命名

- 用人能读的字符串，便于审阅图风格。
- 影视：`<剧名拼音>` 或 `<剧名拼音-集号>`：`changjiang-ruhai` `nanling-yexing-ep01`
- 人物：`<姓拼音-单字>`：`zhou-ning` `chen-bailu`
- 通用：`hero-1` `rail-3` `mood-relax`
- **不要用 `image-1` `pic-test`**——会让本地化 CREDITS 看起来很可笑。

## 9. 长度上限速查

| 字段 | 最大字符 | 备注 |
| --- | --- | --- |
| 主标题（页面 hero） | 8 | 超出会断行或被裁 |
| 卡片标题 | 12 | 12 之后要 ellipsis |
| 副标题 / meta | 16 | 多行需测 |
| DUI 提示按钮文字 | 14 | 14 之后强制换行 |
| 角标 | 4 | 角标超 4 字会破版 |

## 10. 禁止

- Lorem Ipsum / 拼音占位 / 「测试 1」「测试 2」。
- 真实未授权人名 + 真实人脸 + 真实片名 三者同时出现。
- 评分写 `?.?` `--` `0.0`——会让卡片看起来在加载失败。
- 用 emoji 代替图标（除非 Figma 明确标注）。
